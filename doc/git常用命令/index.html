Git基础原理

图源：runoob.com

基础

command 说明
git init git init [project-name] 新建Git仓库
git clone [url] 克隆仓库到本地
git remote add [name] [url] 增加一个远程仓库
git add [file/dir] 添加到暂存区
git commit [file/dir] -m [message] 提交到本地仓库区
git push [remote] [local-branch]:[remote-branch] 将本地local-branch推送到remote仓库的remote-branch分支
git pull [remote] [remote-branch]:[local-branch] 拉取remote仓库remote-branch到本地local-branch分支，并合并
git checkout [branch] 切换到指定分支，并更新工作区

merge

reset

用于回退版本，可以回退到某一次提交的版本。

  1. 指定回退方式【默认--mixed】

    方式 提交点 工作区 暂存区
    --soft 改变 不变 不变
    --mixed 改变 不变 改变
    --hard 改变 改变，丢弃指定版本后的修改 改变

  2. 指定回退版本

    • <commit版本号> ：回退到该commit
    • HEAD^ \ HEAD~1 ：上一个版本
    • HEAD^^ \ HEAD~2 ：上上个版本

  3. 可选择回退的文件【默认整个分支】

rebase

  1. 合并本地n个commit

  2. 分支合并【相较于merge，减少commit次数】

revert

查询提交，不同于reset ：

  • revert 撤销操作是一次反向的commit记录，HEAD继续前移；
  • reset是删除commit，HEAD回退。
